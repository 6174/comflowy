## ComfyUI 介绍

ComfyUI 是一个基于节点的用户界面，用于 Stable Diffusion。ComfyUI 是由 Comfyanonymous 于 2023 年 1 月创建的，他创建了这个工具来学习 Stable Diffusion 的工作原理。

更直观的理解它的价值，StabilityAI，Stable Diffusion 的创建者，使用 ComfyUI 在内部测试 Stable Diffusion。他们后来雇用了 Comfyanonymous 来帮助他们开发内部工具，所以可以看到作者的 Github 主页的组织是 StabilityAI。

ComfyUI 看起来像这样：

基于节点的界面最常见于 3D 设计和 VFX 行业。你可能在 Maya 或 Blender3D 等工具已经遇到过它们。

在这些界面中，每个节点都会执行一些代码。

节点具有输入，这些输入是传递给代码的值，以及输出，这些输出是代码返回的值。

使用鼠标，用户可以：

创建新节点
编辑节点上的参数（变量）
通过它们的输入和输出将节点连接在一起


# ComfyUI vs Stable Diffusion WebUI vs SD.Next

许多新手来自像 AUTOMATIC1111、InvokeAI 或 SD.Next 这样的简单界面。他们都在想同样的问题：为什么要使用这样一个复杂的工具来做同样的事情呢？

流程看起来就很复杂，里边的概念太难理解了，这像是给程序员的工具而不是用户的。

确实，如果只是想要最简单的找个模型，然后执行 PromptText 得到图片，那完全可以使用 Stable Diffusion WebUI。但是如果你想要把 Stable Diffusion 应用到你的真实应用中，或者在项目中真实使用
那可能真的只有 ComfyUI 或者直接使用代码能满足你的需求了，相比与代码，ComfyUI 明显是更高效的方案。


1. 本质逻辑的不同：ComfyUI 本质是把底层的代码逻辑用可视化的方式串联起来而已，逻辑流程都可以用户自定义，这就带来了它的超强扩展性。
2. 性能和速度的不同：因为它的可自定义性，你可以根据你的场景进行流程性能优化，这在 Stable Diffusion 中是无法做到的。
3. 真正的理解 Stable Diffusion：如果你真的想成为专业的从业者，而不是业余爱好，那你就应该要理解它背后的执行逻辑，而 ComfyUI 能帮助你理解每个细节过程。

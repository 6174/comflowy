简单理解，Comflowy 是基于 ComfyUI 进行二次开发的产品，其内核依然是 ComfyUI。并且我们也按照 ComfyUI 的协议要求，对代码进行了开源。

那这就意味着，ComfyUI 几乎所有的功能，Comflowy 都有。同时我们团队还优化了 ComfyUI 的各种体验问题，使得其更加易用，同时功能也更加强大。

## Comflowy 特色功能

### 1. 云端运行

我们提供的云端版本：

1. 可以让你直接使用 ComfyUI，不需要顾虑前面一章提到的环境问题。
2. 预装了超过 100 个插件，几乎涵盖市面上的常用节点。
3. 可直接使用 Civitai 上的社区模型，无需下载。
4. 支持高性能的显卡，最高支持 80G 显存的 A100，不需要担心显存不足导致无法运行的问题。

### 2. 丰富的模板

正如我在前一个章节中提到的，ComfyUI 就像是乐高积木，你需要看说明书，才能拼出你想要的形状。而模板以及本教程，就是帮助你快速上手 ComfyUI 的说明书。

### 3. 丰富的 API 节点

目前市面上除了开源的模型外，还有不少是闭源模型，比如能生成有文字的图片的 Recraft 模型，亦或者是能生成视频的 Kling 和 MiniMax 模型。

这些模型都需要通过 API 的方式来调用，大部分人不会写 API 代码，所以只能购买他们的会员，并在他们的产品里使用这些模型。

这样一来产品众多，如果每个产品都去购买会员，成本就太高了，而且大概率你也用不完每个月的配额。二来，如果仅能在他们的产品里使用，限制也比较大。比如你想要使用 A 模型生成图片，然后让 B 模型使用 A 模型生成的图片，进行图生视频，这种操作就比较繁琐。

所以为了解决这个问题，我们还额外提供了 API 节点，可以让你直接在 ComfyUI 中调用这些模型，同时仅需在 Comflowy 中购买一次会员即可使用市面上大多的模型，总费用会少很多。

### 4. 支持应用模式

Comflowy 除了支持像 ComfyUI 那样的工作流模式外，还支持将工作流转为应用，这样你就不用去关注繁琐的工作流，而是像使用熟悉的应用那样，使用该工作流。同时我们还支持分享功能，你可以将你的应用分享给其他 Comflowy 订阅用户使用。

## Comflowy 目前的不足

说了那么多优势，Comflowy 当然也有一些不足之处。我希望各位在订阅之前，先了解这些不足之处，再做付费的决定。

### 1. 不能自定义安装插件

因为运行 ComfyUI 需要依赖大量的插件，而插件的安装和运行，需要安装其依赖的环境。为了保证整个环境的稳定性，我们无法让你自定义安装某些插件。你只能用我们装好的插件。

另外，我们已经预装了超过 100 个插件，几乎涵盖市面上的常用节点，所以你基本不需要自定义安装插件。

如果你确实需要自定义安装插件，可以加入我们的 [Discord](https://discord.gg/cj623WvcVx) 交流群，或者点击产品右下角的客服按钮，向我们反馈。我们收到反馈后，会评估该插件是否能支持。如果可以支持，我们会尽快安装支持。

所以我建议你在订阅之前，先导入一些 ComfyUI 工作流，看看是否能正常导入。再做决定。

### 2. 部分插件的适配可能有问题

有部分插件的适配会有问题，不过我们也在逐步完善中。

如果你在使用的过程中，遇到了插件无法使用的问题，欢迎加入我们的 [Discord](https://discord.gg/cj623WvcVx) 交流群，我们会尽力帮助你。也可以点击产品右下角的客服按钮，向我们反馈。我们目前正在全力适配各种插件。
